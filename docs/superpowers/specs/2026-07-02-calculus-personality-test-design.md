# 高数解题人格测试网站设计

## 背景

项目要把现有的高数主题人格测试内容做成一个可正式发布、同时有作品感的静态网站。题目、分支规则、计分维度、16 种人格结果和角色图已经在项目文件夹中提供。

现有内容来源：

- `高数解题人格测试.md`：题目、变量、计分方式、结果描述、输出模板。
- `问题分支Map.md`：题目分支、状态变量传递、最终计分和平分处理。
- `personality_characters_d_route/`：16 张人格角色图和总览图。

## 目标

构建一个 React/Vite 静态单页网站。用户能从作品化首页开始测试，完成 8 题后获得人格证书，并保存精简证书卡或完整报告图。

网站不评价数学能力，只呈现“面对复杂问题时如何进入、判断、推进和取舍”的思维风格。

## 已确认决策

- 使用 React/Vite，构建结果仍是静态站点。
- 视觉方向为“手绘学术工坊”。
- 色系为“薄荷灰 + 铅笔紫”，配合角色图中的红、绿、黄等点缀色。
- 首页使用“笔记本开场”：像翻开一本高数手账或人格题册。
- 答题页使用“聚焦题卡”：一屏一题，题干和选项是主角。
- 答题过程中完全不显示维度倾向，只显示进度。
- 结果页使用“双层证书”：
  - 首屏展示精简人格证书。
  - 下方展开完整报告。
- 分享产物提供两种：
  - 默认精简证书卡。
  - 完整报告图。
- 题目和结果文案允许轻微润色，但不改变含义、计分或分支逻辑。
- 先做静态版，不接后端统计；数据结构保留后续加统计的余地。

## 用户体验

### 首页

首页第一屏应该直接建立“高数人格测试”的主题，而不是做营销式落地页。

布局采用笔记本开场：

- 左侧或上方展示网站标题、简短说明和开始按钮。
- 右侧或下方展示角色拼贴、草稿纸、公式贴纸、几何符号等轻装饰。
- 移动端保持单列，标题、说明和开始按钮优先出现，角色图作为辅助视觉。

首页文案基调应轻松、可信，不夸张：

- 强调这是趣味人格测试。
- 说明测试不评价数学好坏。
- 说明大约 8 题完成。

### 答题流程

答题页采用聚焦题卡：

- 顶部显示题号和进度，例如 `03 / 08`。
- 中部显示当前题目标题、题干和 4 个选项。
- 底部提供返回上一题和继续逻辑。
- 不显示任何维度分数、倾向标签或内部计分词。

题目必须根据前序状态改变题干：

- Q1 设置 `T`，决定 Q2 版本。
- Q2 设置 `M`，决定 Q3 题干。
- Q3 设置 `K`，决定 Q4 题干。
- Q4 设置 `F`，决定 Q5 题干。
- Q5 设置 `P`，决定 Q6 题干。
- Q6 设置 `R`，决定 Q7 引导语。
- Q7 设置 `G`，决定 Q8 题干。
- Q8 后进入计分和结果。

用户返回上一题并改选时，后续答案和状态必须被重新计算或清空，避免旧分支污染新结果。

### 结果页

结果页先揭晓精简人格证书，再提供完整报告。

精简证书包含：

- 人格角色图。
- 中文人格名和短称，例如 `创新整合型｜创构者`。
- 中文代码和字母代码，例如 `式-构-统-巧 / L-X-W-F`。
- 一句短描述。
- 四个维度关键词结果。
- 网站名或测试名。

完整报告包含：

- 完整结果描述。
- 四组维度分布分数。
- 对外维度说明：
  - 信息入口：情境型 C / 结构型 L。
  - 决策依据：依据型 R / 探索型 X。
  - 处理方式：分步型 P / 整合型 W。
  - 行动偏好：稳健型 B / 灵活型 F。
- 保存精简证书卡按钮。
- 保存完整报告图按钮。
- 重新测试按钮。

### 分享卡

分享卡由前端生成图片，不依赖后端。

精简证书卡适合社交分享：

- 信息少、角色图突出。
- 短描述最多一到两行。
- 中文排版必须清楚，不拥挤。

完整报告图适合保存：

- 包含完整描述摘要和维度分布。
- 尺寸可以比精简卡更长。
- 文字应分段排布，避免一整块长文本。

## 视觉设计

### 色系

主色系：

- 薄荷灰：用于背景、浅色面板和温和提示。
- 铅笔紫：用于主要按钮、选中态、证书重点信息。

辅助色：

- 珊瑚红、黄赭、松绿、铅蓝作为少量点缀，呼应角色图。

约束：

- 页面不能被单一紫色或米色支配。
- 正文必须使用足够深的颜色。
- 装饰色不能压过题干和选项。

### 排版

文字排版是一级约束。

- 答题页题干需要足够行距，长句应自然换行。
- 选项使用单列布局，尤其移动端不能使用窄两列。
- 结果描述分段展示，避免长段落压迫。
- 分享图中长文本需要限制行数或拆分摘要。
- 按钮文字不能溢出。

### 图片

16 张角色图是结果页和分享卡的核心资产。

- 首页可以使用总览图或若干角色图做拼贴。
- 答题页不强依赖角色图，避免抢阅读焦点。
- 结果页必须匹配对应人格角色图。

## 数据模型

题库数据应结构化维护，不从 Markdown 运行时解析。

核心数据包括：

- `questions`：每题定义、动态题干、选项、计分标签和状态变量设置。
- `dimensionPairs`：四组二元维度。
- `tieBreakRules`：四组平分回看规则。
- `results`：16 种人格的代码、名称、短称、描述、图片路径和短描述。

内部计分标签：

- `景` 对应 C。
- `式` 对应 L。
- `判` 对应 R。
- `构` 对应 X。
- `拆` 对应 P。
- `统` 对应 W。
- `稳` 对应 B。
- `巧` 对应 F。

最终结果按四组比较得到：

- `景 / 式` 得到 C 或 L。
- `判 / 构` 得到 R 或 X。
- `拆 / 统` 得到 P 或 W。
- `稳 / 巧` 得到 B 或 F。

平分处理必须按原规则：

- `景 / 式` 平分：看 Q1，其次 Q7。
- `判 / 构` 平分：看 Q2，其次 Q8。
- `拆 / 统` 平分：看 Q4，其次 Q7。
- `稳 / 巧` 平分：看 Q6，其次 Q8。

如果回看题目没有给出该维度的有效倾向，应使用该维度的第一个类别作为稳定兜底，并在测试中覆盖这种情况。

## 技术设计

使用 React/Vite 构建静态单页应用。

建议模块边界：

- `src/data/testData.ts`：题库、分支、结果和图片映射。
- `src/lib/scoring.ts`：计分、平分处理、结果代码生成。
- `src/lib/shareImage.ts`：保存精简证书卡和完整报告图。
- `src/components/Home.tsx`：首页。
- `src/components/QuizCard.tsx`：答题卡。
- `src/components/ResultCertificate.tsx`：结果证书。
- `src/components/ReportDetails.tsx`：完整报告。
- `src/App.tsx`：页面状态和流程。

分享图片生成可以优先使用 DOM 转图片方案。若引入依赖，应保持静态构建可用，并验证中文字体和图片跨域问题。所有图片来自本地项目资源，避免外链资源导致保存失败。

## 错误和边界

- 用户未完成全部题目时不能进入结果页。
- 用户返回并改选时，后续答案需要清空或重算。
- 图片加载失败时显示人格名称和代码，不让页面空白。
- 保存图片失败时提供复制结果文字作为降级方案。
- 移动端窄屏必须保持题干和选项可读。

## 测试与验收

功能验收：

- 8 题流程可完整走通。
- Q2 到 Q8 的动态题干正确跟随状态变量。
- 四维分数累计正确。
- 平分规则正确。
- 16 种人格都能匹配到结果和角色图。
- 返回上一题改选后，结果按新路径计算。
- 精简证书卡和完整报告图可保存。

视觉验收：

- 桌面端和移动端无文字溢出。
- 答题页选项不拥挤。
- 结果证书首屏信息清楚。
- 分享图中文字不重叠、不截断核心信息。
- 色系保持薄荷灰 + 铅笔紫为主，但不形成单一紫色页面。

## 非目标

本阶段不做：

- 后端统计。
- 登录或用户系统。
- 排行榜或结果分布页。
- 社交平台直接发布。
- 多语言版本。
